Đồ án Thiết kế 1 — ĐTVT HUST

Hệ thống giám sát hình ảnh từ xa IoT

ESP32 + OV7670 + TFT Display + Web Server — Kế hoạch chi tiết & Phân tích khả thi

✅ Đây có phải đồ án IoT không?
CÓ — 100% là một dự án IoT (Internet of Things) chuẩn mực.
Dự án này thỏa mãn tất cả 5 tiêu chí của một hệ thống IoT hoàn chỉnh:

📡 1. Sensing (Thu thập)

ESP32 #1 + OV7670 đóng vai trò Edge Sensor Node — thu thập dữ liệu hình ảnh từ môi trường thực.

🌐 2. Connectivity (Kết nối)

Truyền dữ liệu qua WiFi — giao thức truyền thông không dây chuẩn IoT. Có thể mở rộng MQTT/HTTP.

☁️ 3. Cloud/Server (Đám mây)

Web Server lưu trữ ảnh, database quản lý dữ liệu — đúng mô hình Edge → Cloud → Client.

🖥️ 4. Processing (Xử lý)

Server xử lý, lưu trữ, phân phối ảnh. Có thể mở rộng thêm AI nhận diện trên server.

🎛️ 5. Actuation (Điều khiển)

ESP32 #2 + TFT là Client Node — hiển thị ảnh và điều khiển từ xa camera node.

⚠️ Lưu ý về TFT 1.8": Hầu hết module TFT 1.8" (ST7735) không có cảm ứng. Nếu bạn cần điều khiển, có 2 cách:
• Cách 1 (Rẻ): Dùng 3–4 nút nhấn vật lý bên cạnh màn hình (lên/xuống/chọn/back) → giao diện menu đơn giản.
• Cách 2: Đổi sang TFT 2.4" ILI9341 + cảm ứng XPT2046 (~70k) → có touch thật sự. Nhưng thêm chi phí.
→ Khuyến nghị Cách 1 để giữ chi phí thấp và vẫn đủ chức năng cho ĐA1.
🔍 Phân tích khả thi (Feasibility Analysis)
Hạng mục Đánh giá Chi tiết Rủi ro
OV7670 (no FIFO) + ESP32 Trung bình Driver esp32-camera hỗ trợ OV7670. Không FIFO thì phải dùng I2S camera mode. Resolution tối đa ~QQVGA (160×120) nếu không có PSRAM.
Truyền ảnh qua WiFi Khả thi QQVGA Grayscale = ~19KB/frame. Qua WiFi HTTP tốc độ ~1-3 MB/s → truyền được ~50-150 frame/s lý thuyết. Thực tế: 2-5 fps.
TFT 1.8" hiển thị ảnh Dễ ST7735 SPI, 160×128 pixel. Thư viện TFT_eSPI rất ổn định. Hiển thị ảnh RGB565 rất nhanh (DMA SPI).
Web Server + Database Dễ Dùng Python Flask/FastAPI + SQLite hoặc Firebase. REST API nhận/trả ảnh. Bạn đã có kinh nghiệm Python.
Điều khiển từ xa Khả thi Display Node gửi lệnh JSON → Server → Camera Node polling lệnh. Độ trễ ~0.5-2s (chấp nhận được).
Resolution & chất lượng ảnh Hạn chế OV7670 không FIFO + ESP32 không PSRAM → QQVGA 160×120. Đủ demo nhưng không sắc nét. Nếu có PSRAM: QVGA 320×240.
KẾT LUẬN: Dự án KHẢ THI — Rủi ro chính nằm ở việc OV7670 không FIFO có thể gây khó khăn ban đầu. Nhưng driver esp32-camera đã hỗ trợ nên hoàn toàn làm được. Các phần còn lại đều có thư viện tốt và bạn đã có kinh nghiệm.

⚙️ Chức năng chi tiết từng Node

📷 Camera Node (ESP32 #1 + OV7670)

  • Khởi động camera OV7670 qua SCCB (I²C)
  • Chụp ảnh liên tục (QQVGA 160×120, Grayscale hoặc RGB565)
  • Kết nối WiFi đến router
  • Gửi ảnh lên Web Server qua HTTP POST (Base64 hoặc binary)
  • Polling lệnh điều khiển từ Server (chụp/dừng/đổi mode)
  • LED status: đang chụp / đang gửi / lỗi

🖥️ Display Node (ESP32 #2 + TFT 1.8")

  • Kết nối WiFi đến cùng router
  • Polling ảnh mới nhất từ Web Server (HTTP GET)
  • Hiển thị ảnh lên TFT ST7735 (SPI, 160×128)
  • Giao diện menu điều khiển bằng 4 nút nhấn:
      [▲] Chọn lên   [▼] Chọn xuống
      [OK] Xác nhận   [◀] Quay lại
  • Chức năng: Chụp ảnh / Xem gallery / Đổi chế độ / Cài đặt WiFi
  • Hiển thị timestamp, trạng thái kết nối

☁️ Web Server + Database

Backend (Python Flask/FastAPI)

  • POST /api/upload — Camera Node gửi ảnh lên
  • GET /api/latest — Display Node lấy ảnh mới nhất
  • GET /api/gallery — Lấy danh sách ảnh đã chụp
  • POST /api/command — Display Node gửi lệnh điều khiển
  • GET /api/command — Camera Node polling lệnh mới
  • Lưu ảnh lên thư mục server + metadata vào SQLite

Frontend (Web Dashboard — bonus)

  • Trang web xem ảnh trực tuyến (HTML/CSS/JS)
  • Gallery ảnh đã chụp với timestamp
  • Nút điều khiển camera từ xa
  • Trạng thái kết nối real-time
  • Responsive (xem được trên điện thoại)

📅 Kế hoạch thực hiện theo tuần (8 tuần)
Tuần 1 — Chuẩn bị & Nghiên cứu

🔧 Setup môi trường + Nghiên cứu datasheet

  • Cài Arduino IDE + ESP32 Board Manager (hoặc PlatformIO)
  • Đọc datasheet OV7670 (register map, timing diagram, SCCB protocol)
  • Đọc ESP32 Technical Reference (I2S Camera Mode, GPIO matrix)
  • Flash chương trình LED Blink lên ESP32 để xác nhận board hoạt động
  • Xác nhận ESP32 là WROOM hay WROVER (kiểm tra PSRAM)
  • Deliverable: ESP32 flash được code, hiểu pinout OV7670
Tuần 2 — Nối dây & Test Camera (QUAN TRỌNG NHẤT)

📷 Kết nối OV7670 ↔ ESP32 #1, chụp frame ảnh đầu tiên

  • Nối dây OV7670 → ESP32 theo pinout (16 dây: D0-D7, PCLK, VSYNC, HREF, XCLK, SDA, SCL, RESET, PWDN)
  • Cài thư viện esp32-camera, cấu hình cho OV7670
  • Flash firmware test: chụp 1 frame QQVGA (160×120) Grayscale
  • Gửi ảnh qua Serial (hex dump) hoặc WiFi HTTP → xem trên PC
  • Debug nếu ảnh bị lỗi (sai màu, sọc, trắng đen → thường do timing hoặc sai register init)
  • Deliverable: Chụp được 1 frame ảnh rõ ràng từ OV7670 ✅
Tuần 3 — WiFi + Gửi ảnh lên Server

🌐 Kết nối WiFi, dựng Server, truyền ảnh

  • Viết firmware ESP32 #1: kết nối WiFi → chụp ảnh → HTTP POST lên server
  • Dựng Web Server đơn giản bằng Python Flask:
      • POST /api/upload nhận ảnh binary, lưu file + SQLite
      • GET /api/latest trả ảnh mới nhất
  • Test: ESP32 chụp → gửi → server lưu → mở trình duyệt xem ảnh
  • Deliverable: Ảnh từ camera xuất hiện trên web browser ✅
Tuần 4 — Display Node (ESP32 #2 + TFT)

🖥️ Kết nối TFT 1.8", hiển thị ảnh

  • Nối TFT ST7735 → ESP32 #2 (6 dây SPI: SCK, MOSI, CS, DC, RST, BL)
  • Cài thư viện TFT_eSPI, test hiển thị text + hình cơ bản
  • Viết firmware: ESP32 #2 kết nối WiFi → GET ảnh từ server → decode → hiển thị TFT
  • Nối 4 nút nhấn, viết code đọc nút (debounce)
  • Deliverable: TFT hiển thị được ảnh nhận từ server ✅
Tuần 5 — Điều khiển từ xa

🎛️ Giao diện menu + Gửi lệnh điều khiển

  • Thiết kế giao diện menu trên TFT (dùng nút nhấn điều hướng):
      • Menu chính: [📸 Chụp ảnh] [📁 Gallery] [⚙ Cài đặt]
      • Gallery: xem lại ảnh đã chụp (lướt trái/phải)
  • API điều khiển: Display Node → POST /api/command → Server lưu → Camera Node polling GET /api/command
  • Lệnh hỗ trợ: capture (chụp), stream_on/off, set_resolution, restart
  • Deliverable: Nhấn nút trên TFT → Camera chụp ảnh → ảnh hiện trên TFT ✅
Tuần 6 — Web Dashboard + Database

☁️ Hoàn thiện Backend + Frontend web

  • Hoàn thiện database (SQLite): bảng images (id, filename, timestamp, size, source)
  • Viết web dashboard (HTML/CSS/JS): gallery ảnh, xem ảnh real-time, nút điều khiển
  • Thêm trang thống kê: số ảnh đã chụp, uptime camera, trạng thái kết nối
  • Test toàn bộ hệ thống end-to-end: Camera → Server → TFT + Web
  • Deliverable: Web dashboard hoạt động, database lưu trữ ổn định ✅
Tuần 7 — Tối ưu & Kiểm thử

🔬 Đo đạc, tối ưu, xử lý lỗi

  • Đo FPS thực tế, latency truyền ảnh, độ ổn định WiFi
  • Tối ưu: nén ảnh (JPEG trên server), giảm polling interval, retry khi mất kết nối
  • Xử lý edge case: mất WiFi, server down, camera treo → auto-reconnect
  • Test stress: chạy liên tục 1-2 giờ, kiểm tra memory leak
  • Deliverable: Hệ thống chạy ổn định, có số liệu đo đạc ✅

⚠️ Rủi ro & Phương án dự phòng

🔴 Rủi ro 1: OV7670 không chụp được ảnh

Nguyên nhân: Sai dây, sai register init, timing không khớp.

Dự phòng: Thử dùng code Arduino cơ bản (không qua esp32-camera) để đọc raw data bằng I2S trực tiếp. Nếu vẫn lỗi → mua module ESP32-CAM (~75k, đã tích hợp OV2640, chắc chắn hoạt động) thay thế Camera Node.

🟡 Rủi ro 2: Ảnh chất lượng thấp, nhiều nhiễu

Nguyên nhân: OV7670 không FIFO → mất pixel khi I2S overrun.

Dự phòng: Giảm resolution xuống QQVGA (160×120), dùng Grayscale thay RGB. Đảm bảo XCLK cho OV7670 đủ 20MHz (dùng LEDC PWM).

🟡 Rủi ro 3: Truyền ảnh chậm, lag nhiều

Nguyên nhân: HTTP overhead, WiFi congestion.

Dự phòng: Chuyển sang UDP socket (nhanh hơn HTTP). Hoặc nén ảnh trên server trước khi gửi cho Display Node. Giảm tần suất chụp.

🟢 Rủi ro 4: Hết GPIO cho TFT + nút nhấn

Nguyên nhân: ESP32 #2 cần GPIO cho SPI (TFT) + 4 nút.

Dự phòng: TFT dùng 5 chân SPI + 4 nút = 9 GPIO. ESP32 DevKit có ~25 GPIO khả dụng → dư thừa, không phải vấn đề.
